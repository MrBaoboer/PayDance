# 设置页首次启动向导按钮对齐设计

## 目标

Web Preview 设置面板中，“首次启动向导”标题与“打开首次启动向导”按钮位于同一行；按钮右边缘与下方 GitHub 按钮右边缘保持一致。桌面端不展示该入口，现有点击行为、文案和 GitHub 区域保持不变。

## 方案

复用 `SettingsGroup` 的 `action` 插槽。该插槽已经为“开机自动启动”提供标题行两端对齐能力，因此无需为向导卡片新增布局容器或绝对定位。

`SettingsOnboardingAction` 保持单一按钮根节点，并增加一个响应式右侧外边距。该外边距等于 GitHub 右侧列宽与 GitHub 按钮宽度之差的一半，从而复现 GitHub 按钮在其列中居中产生的右侧留白，保证不同设置面板宽度下两者右边缘一致。

## 响应式与国际化

- 标题区域使用现有 `space-between` 和间距变量，中文、英文均可自然收缩。
- 按钮文字不固定宽度，避免英文文案被压缩。
- 边缘偏移使用与 `SettingsAboutFooter` 相同的 `clamp()` 宽度表达式，不引入仅适配截图尺寸的固定坐标。

## 验证

- 组件测试确认标题和按钮共享 `group-title--split` 标题行。
- 源码回归测试确认右侧偏移与 GitHub 列、按钮宽度公式一致。
- 运行设置组件测试、完整快速验证和 Web Preview 视觉 QA。
- 在本地 Web Preview 的目标设置面板尺寸下检查同一行与右边缘对齐。
