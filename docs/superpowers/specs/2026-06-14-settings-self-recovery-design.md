# 设置按项自修复设计

**目标：** 本地设置异常时，薪跳在后台只修复出错的项目，保留其余有效设置，并且不显示长期“配置已损坏”提示。

## 行为

- 单个工资、工作日、时间或开关值无效时，只恢复该项的默认值。
- 工作开始与结束时间互相冲突时，只恢复这两个关联项。
- 午休时间超出工作时段时，只恢复午休开始、结束和开关这一最小关联组。
- 来自未来版本的配置仍读取当前版本认识且有效的字段，忽略未知字段。
- 整个设置文件无法读取时，恢复全部默认值。
- 每次修复后立即写回当前版本配置，避免下次启动重复修复。
- 设置中心和新手引导不展示恢复黄条，也不因恢复行为自动打开设置中心。
- 保存设置真正失败时，继续保留现有错误提示。

## 验证

- 迁移单元测试覆盖单项、工作时间组、午休组和未来版本配置。
- 设置加载测试覆盖自动写回、全量回退和无恢复提示。
- 组件源码测试确保恢复提示属性、文案和样式均被删除。
