# DESIGN

## 设计原则

- 安静、克制、清爽，不追求炫技。
- 主视觉只服务金额跳动，不添加会抢注意力的装饰。
- 避免灰、脏、廉价的质感；深色模式尤其要保持干净的黑色层次。
- 主界面层级最多两层：金额层与仪表盘层，避免继续叠加小卡片。

## 信息层级

1. 今日入账金额。
2. 已工作、距离下班或距离上班、今日预计。
3. 今日进度条。
4. 薪资说明和设置等低频入口。

## 窗口策略

- 主窗口保持大圆角无边框，原生窗口阴影应避免污染四角。
- 顶部状态与操作按钮需要留出足够内边距，不贴边。
- 首次启动向导允许拖拽和调整大小，但面板本身应更安静、更不透明。
- 迷你悬浮窗口只呈现金额和透明度体验，不新增复杂控件。

## 主题 Token

- 文本主色使用干净的近黑或近白。
- 重点色使用橙色收入线索，但只用于状态点、进度、焦点和必要强调。
- 仪表盘背景可有轻微半透明质感，但不得出现脏灰泛白。
- 进度条轨道保持平面、干净；圆点可以有克制光感。

## 字体与数字

- 主金额数字字体不随意更换。
- 仪表盘、设置中心、薪资说明和首次向导中的数字、英文、符号统一使用仪表盘数字字体。
- `h`、`m`、`¥` 与数字之间必须保留可感知间距，避免粘连。

## 动效边界

- 金额变化可以有克制脉冲光，范围可感知但持续时间短。
- 不添加第二层环境光或大面积发光。
- 主题切换应同步 Web 与原生窗口主题，避免四角或边框短暂错色。

## 测试要求

- 薪资计算、夜班边界和设置迁移优先使用纯函数测试。
- 关键 UI 交互优先使用 Vue Test Utils 行为测试。
- 源码字符串测试只保留少量发布约束、视觉防回退和架构边界断言。
- 后续深挖 UI 时补充浅色、深色、首次向导、迷你悬浮的截图回归测试。
