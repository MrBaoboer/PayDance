# Governance

PayDance is a solo maintainer project. The maintainer makes final decisions on
product scope, release timing, licensing, trademarks, and whether a contribution
fits the project.

## Decision Principles

- Keep PayDance a local-first desktop real-time salary dashboard.
- Prefer small, testable changes over broad rewrites.
- Protect user trust: no accounts, no telemetry, no salary data upload by
  default.
- Keep Windows release quality ahead of new platform breadth.
- Do not expand into time tracking, attendance, payroll, personal finance, or
  notification-heavy workflows without a deliberate product decision.

## Contribution Path

1. Open or claim an issue before larger changes.
2. Use `good first issue` and `help wanted` labels for low-risk community work.
3. Follow [.github/CONTRIBUTING.md](.github/CONTRIBUTING.md).
4. Include verification evidence in the pull request.

## Maintainer Authority

The maintainer may close issues or pull requests that are out of scope, unsafe,
too broad, insufficiently verified, legally unclear, or likely to increase
maintenance burden beyond current capacity.

## Changes to Governance

Governance changes should be made through pull requests and should stay concise.
If PayDance gains additional regular maintainers, this file should be updated
before those maintainers receive release or repository administration authority.
